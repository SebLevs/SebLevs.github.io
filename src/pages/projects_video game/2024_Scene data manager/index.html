This project is an answer to the hassle that managing scenes can be in big projects under Unity Engine.

Scene Data Manager is a tool that allows for an easy management of scenes through scriptable objects called scene data.

Data
A scene asset usable from anywhere

Upon creating a new scene asset, a scene data scriptable object is also created at the same path and the scene asset is automatically linked to it.

The scene data can thereafter be used as a proxy for the scene asset, allowing for easy access to the scene asset from anywhere in the project.

The base core scenes (Main menu, Persistent, Loading) are also available to the user through a singular scriptable object.

Visual
Following the UI scheme of Unity

The UI of the tool is simple and intuitive, following the UI scheme of Unity and the Gestalt laws.

Programming
Custom dropdown implementation

The default editor dropdown was lacking in both visual and code features.
I decided that it would be best if I create my own while using some elements from Unity default like the listview element.

In details:

  • If the user types in the search bar, the dropdown will find every header that contains the input and populate the dropdown listview with the results.
  • The dropdown position is absolute and automatically position itself as a child of the RootVisualContainer element in the hierarchy.
    The dropdown will therefore always be visible to the user, wherever it's owner is originally placed in the hierarchy.
  • The dropdown is composed of buttons.
  • Each button is associated with an element of the list view.
  • A custom behaviour may be dictated by the programmer for the click event of any of the dropdown elements.

Better visual element initialization and extension methods

Extension methods were used to allow an easier initialization of VisualElements through the reference of UXML and USS files.
This provides a cleaner code and easier maintenance.

Extension methods were also implemented to facilitate specific actions related to visual elements.

Making the usage of listviews more intuitive

The listview element is a powerful tool that allows for the virtualization of visual elements.
However, it is not the most intuitive to use.
I decided to create a utility class for the listview element that would make it easier to use.

Given a list to bind to, adding and binding elements to the listview is now managed automatically by the utility class.

Behaviours may be added to the utility class in the future, as the need arises.

Quickly open and load scenes

A quick load button as well as a quick open button are available and always visible to the user.

The buttons are centered and placed above the reference name of the loader scene data for an intuitive usage.
